Моя Front-End сборка

Февраль 1, 2018

Сборка осуществляется с помощью gulp.

Gulp - инструмент сборки веб-приложения, позволяющий автоматизировать повторяющиеся задачи, такие как сборка и минификация CSS- и JS-файлов, запуск тестов, перезагрузка браузера и т.д.

Используемые мной возможности:

  • Создание исходных файлов
  • Компиляция jade и sass файлов
  • Вывод wiredep-ом стилей и скриптов из Bower
  • Build стилей и скриптов
  • Компиляция js и css
  • Минификация js и css
  • Сжатие изображений
  • Кэширование скриптов и стилей
  • Запуск сервера
  • Live перезагрузка и очистка
  • Работа с svg

Инструменты:

  • NODE.JS. Download for windows.
  • GIT. Download for windows.
  • RubyInstaller. Download for windows.
  • ConeMu терминал.

Технологии:

  • JADE - препроцессор HTML и шаблонизатор.
  • SASS - CSS препроцессор.
  • npm - пакетный менеджер node.js.
  • Bower - пакетный менеджер для web.

Менеджеры пакетов упрощают установку и обновление зависимостей проекта, то есть сторонних библиотек, которые он использует: jQuery, др.

Главное отличие npm и Bower — подход к установке зависимостей пакетов.

npm устанавливает зависимости для каждого пакета отдельно, в итоге получается большое дерево пакетов (node_modules/grunt/node_modules/glob/node_modules/…), где может быть несколько версий одного и того же пакета.

В Бовере каждый пакет устанавливается один раз (jQuery всегда будет в папке bower_components/jquery, сколько бы пакетов от него не зависело) и в случае конфликта зависимостей, Бовер просто не станет устанавливать пакет, не совместимый с уже установленными.

Создание исходников 

mkdir www
cd !$
mkdir app/{css,js} –p
cd app
touch index.jade js/{main.js,plugins.js}
cd ..
ls

!$ - переход во вновь созданную директорию.
Ключ -p - создание самой директории app.
Переход в директорию - cd app.
Переход на уровень выше - cd .. .
Просмотр файлов - ls.
Просмотр скрытых файлов и с точкой - ls -a.
Полностью удалить директорию - rm -r app.
